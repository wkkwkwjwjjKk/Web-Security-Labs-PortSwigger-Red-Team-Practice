# PortSwigger Labs – Web Security Writeups

Este repositório contém meus estudos práticos de Web Security através da PortSwigger Web Security Academy.

O foco é desenvolver habilidades em exploração de vulnerabilidades reais em aplicações web, com ênfase em segurança ofensiva e análise de falhas.

---

## 🎯 Objetivo

Construir base prática em Web Exploitation e preparação para atuação em Red Team.

---

## 📚 Áreas de estudo

### 🔐 Access Control
- User ID controlled by request parameter  
- User ID controlled by request parameter with password disclosure  
- User ID controlled by request parameter with data leakage in redirect  
- User ID controlled by request parameter with unpredictable user IDs  
- URL-based access control can be circumvented  
- Method-based access control can be circumvented  
- Referer-based access control  
- Insecure direct object references (IDOR)  
- User role can be modified in user profile  
- Unprotected admin functionality  
- Unprotected admin functionality with unpredictable URL  
- Multi-step process with no access control on one step  

---

### 💉 Command Injection
- Basic Command Injection  
- Command Injection (delay-based)  
- Blind OS Command Injection (output redirection)  
- Blind OS Command Injection (out-of-band interaction)  
- Blind OS Command Injection (out-of-band exfiltration)  

---

### 📤 File Upload Attacks
- Remote Code Execution via Web Shell Upload  
- Content-Type restriction bypass  
- Path traversal upload  
- Extension blacklist bypass  
- Obfuscated file extension bypass  

---

## 🧠 Habilidades desenvolvidas
- Identificação de vulnerabilidades web  
- Exploração de falhas de controle de acesso  
- Execução remota de comandos (RCE)  
- Técnicas de bypass em validações de segurança  
- Documentação técnica de vulnerabilidades
